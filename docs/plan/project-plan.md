# react-zslj 项目方案 —— 《占事略決》在线排盘与 AI 分析

> 依据：`docs/book/senji-ryakketsu.md`（全书合订本，2026-07-05 采集自 miko.org）
> 调研：`docs/research/similar-projects.md`
> 状态：本方案已实施——排盘引擎独立为 [`zslj-ts-lib`](https://github.com/hackninety/zslj-ts-lib) 库，前端接入库并保留典籍页与 AI 分析。下文为最初的设计方案，保留作背景参考。

## 1. 项目定位

把安倍晴明《占事略決》（现存唯一晴明真撰，约天元六年/983 年）做成**忠实古法的六壬式占在线排盘 + AI 断课**：

1. **古法引擎**：完全按原书三十六法起课（课用九法、天一治法、卅六卦），不混入明清通行体系的后世规则——这是全网没有人做过的东西；
2. **原文可溯**：盘面每个结果（三传取法、天将布局、课体判定）都能点开对应的原文条文；
3. **AI 分析**：把课盘 JSON + 命中的原文条文喂给模型，让 AI"依经断课"而非自由发挥；
4. **对照模式**（可选）：并排显示通行体系（liuren-ts-lib/mingyu-core）的结果，高亮古今分歧。

## 5. 引擎设计：从原文到代码

### 5.1 排盘管线

```
输入(公历时刻 | 四柱) ─→ calendar: 四柱/旬空/月将/旦暮
  ─→ tianpan: 月将加占时布天盘
  ─→ sike: 日干寄宫(第八章)立四课
  ─→ sanchuan: 课用九法依序判定 → 三传 + 判定路径(每步引用原文)
  ─→ tianjiang: 天一治法布十二将(旦暮 + 顺逆)
  ─→ derive: 遁干/空亡/王相/德财鬼/客筹/行年
  ─→ gua36: 卅六卦匹配 → 课体 + 断语
  ─→ ZsljChart(含 refs[] 原文引用数组)
```

### 5.2 原文规则要点（实现规范）

**四课三传（第一章）**：月将加占时；日上神=一课，一课上神之上=二课，辰上神=三课，三课上神之上=四课；取相克者为用。

**课用九法（第二章）**——严格按原文顺序：
1. **贼克**：下贼上优先，无则上克下；
2. **比用**：多课相克时取"与今日比"者——罡日比日取阳神（神后/功曹/天罡/勝先/傳送/河魁＝子寅辰午申戌），柔日比辰取阴神（大吉/大衝/太一/小吉/從魁/徵明＝丑卯巳未酉亥）；
3. **涉害**：俱比/俱不比时"加孟为深、加仲为半、加季为浅"（**不用**后世归本家计克法）；
4. **先举**：涉害俱等时"日为先辰为后、阳为先阴为后"；
5. **遥克**：四课无上下克时，神遥克日优先于日遥克神；俱有取比；俱比再走涉害；
6. **昴星**：无遥克时，罡日取酉上神发用、中传辰上末传日上；柔日取从魁所临下神发用、中传日上末传辰上；
7. **伏吟**：有克仍取克（乙癸日）；无克罡日用日上神、柔日用辰上神；**中传=初传之刑，末传=中传之冲**；
8. **返吟**：有克取克；无克罡日用日冲、柔日用辰冲（丁未/己未/辛未日太一临亥用，丁丑/己丑/辛丑日徵明临巳用）；三传"传辰上，终日上"；
9. **五柔（八专）**：甲寅/庚申/己未/丁未/癸丑五日；有克如常；无克罡日自日上神顺数三神发用、柔日自辰上阴神逆数三神发用，中末"终日辰上"。

**天一治法（第三章）**：
- 旦暮贵人：甲戊庚→丑/未；乙己→子/申；丙丁→亥/酉；（六）辛→午/寅；壬癸→巳/卯；
- **旦=地盘寅~酉时，暮=戌~丑时**（古法分界，区别于通行卯酉界）；
- 顺逆：贵人"常背天门向地户"，原文给出亥/戌/辰/巳四例（亥→子前戌后、戌→酉前亥后、辰→巳前卯后、巳→辰前午后），推广规则在实现时以四例反推验证并做单测锁定。

**卅六卦（第廿六章）**：36 个课体判定器（元首/重审/傍茹/蒿矢/寅视/伏吟/反吟/无淫/狡童逃女/惟薄不修/三交/乱首/龙战/赘婿寓居/阴阳无亲/陞跎/玄胎四牝/联茹/曲直/炎上/稼穑/从革/润下/九丑/天网/无禄/绝纪/五愤四杀/三光三阳/高盖驷马/斲轮织绶/铸印乘轩/斩关/天狱 + 气类物/新故二总卦），一个课盘可命中多卦，全部列出并附断语原文。

**月将双模式**（开关，默认①）：
- ① 中气过宫（天文正确，浦木裕补注"月将者太阳宫也"）；
- ② 字面模式：正月徵明~十二月神后按月建配将（复现古书课例用）。

### 5.3 存疑处的处理原则

原文有歧义的地方（八专"终日辰上"的中末传归属、返吟三传细则、贵人顺逆的推广、"六辛"校勘），一律：**按字面最直接的读法实现 + `options` 开关提供另一种读法 + UI 上标注"文义存疑，见校勘"并链接原文**。不悄悄用通行体系的规则补洞。

### 5.4 测试策略（书自带答案）

第廿六章每卦附有"假令"课例——**30 余条古人给定的标准答案**，如：
- 元首卦：正月甲子日寅时占；重审卦：二月乙巳日午时占；寅视卦：六月戊寅日寅时占；
- 三光三阳卦：六月戊辰日寅时、六月戊寅日寅时；天狱卦：正月庚寅日申时、二月乙酉日巳时……

以"字面月将模式"复现这些课例作为 vitest 黄金用例（月将=月建对应将，时支=课例时辰），凡与假令不符即引擎有错或原文存疑，逐条化解。另与 kinliuren/liuren-ts-lib 交叉验证贼克、比用等古今相同的规则。

## 6. 原文 JSON 化方案（`src/data/book/`）

从 `docs/book/senji-ryakketsu.md` 半自动提取（写一个 `scripts/extract-book.mjs`），结构示例：

```jsonc
// chapters.json 条目
{
  "id": "senji02-3",                    // 章锚点-条序
  "chapter": 2, "title": "課用九法",
  "clause": "第三",
  "text": "若四課俱比、俱不比,以涉害深者為用...",
  "notes": ["涉害,底本做涉客."],        // 校勘记
  "tags": ["三传", "涉害"]
}
// gua36.json 条目
{
  "id": "2603", "no": 3, "name": "元首卦",
  "rule": { "type": "sanchuan", "cond": "一上克下为用" },   // 结构化判定
  "text": "謂以一上剋下為用是也.占事皆以神將,論其憂喜.",
  "jialing": [{ "month": 1, "day": "甲子", "hour": "寅" }]  // 假令→测试用例
}
```

引擎的 `refs.ts` 与 AI 检索共用这套 JSON；`id` 与合订本锚点一致，UI 可跳转原文。

## 7. 排盘 UI（参照 react-liuren 布局约定）

- **顶栏**：项目名「占事略決」+ 引擎/选项切换（月将模式、旦暮界、对照开关）+ 主题切换 + 自定义排盘（公历/四柱 + 真太阳时 + 出生年命/性别）
- **面板序列**（玻璃卡片 + 金色点缀，黑底金字致敬原站气质）：
  1. 基础信息：四柱、月将、旬空、旦/暮、王相状态
  2. **天地盘**：4×4 方盘（外圈十二宫：地盘支/天盘支/天将/遁干，中央嵌四课）；二期加"式盘"圆盘视图（天盘可旋转动画，复刻 `img/01/rikujin01.gif` 的天圆地方式盘）
  3. 四课、三传：三传卡片显示**判定路径**（如"四课无上下克 → 无遥克 → 昴星法·柔日"，每步可点击跳原文第二章对应条）
  4. 十二天将盘：前五后六环列，天一居中概念图（第四章）
  5. 课体（卅六卦）：命中卦列表卡片，卦名 + 所主断语原文 + 假令
  6. 占断助手：选占类（病祟/病死生/产期/产男女/待人/盗失/六畜/闻事/雨/晴，第廿五~卅六章）→ 展示该章原文 + 机器可判部分自动高亮（如"大神加孟不可信"直接给出判定）
  7. 神煞·杂法：日德/日财/日鬼/空亡/十二客/十二筹/行年（含京大本"指年法"插件）
  8. 原文面板：右侧抽屉，任何 ref 点击后定位到合订本对应锚点
  9. JSON 导出 & AI 分析
- **对照模式**：开启后三传/天将并排显示古法 vs 通行体系结果，分歧行高亮并注差异原因（引用 §5.2 差异表）

## 8. AI 分析设计

分三级演进，V1 就有可用价值：

- **V1（剪贴板，零成本）**：一键复制「系统提示 + 课盘 JSON + 命中原文条文 + 占问」的完整提示词。与 react-liuren 的差异在于**自动附带引用**：命中的卅六卦断语、对应占类章节原文、十二将所主、王相状态解释全部随盘打包，模型"依经断课"。
- **V2（应用内直连）**：设置页填 API Key（Anthropic 或任意 OpenAI 兼容端点，localStorage 存储），流式输出；回答中的〔氣類物卦第一〕等引用渲染为可点击标记 → 打开原文面板。默认模型建议 `claude-sonnet-5`（性价比）/`claude-fable-5`（深度）。
- **V3（可选服务端）**：加一层轻代理（Key 不出服务器、缓存、限流），供公开部署用。

提示词骨架：

```
system: 你是研究《占事略決》的六壬式占助手。只依据下面提供的原文条文断课，
        引用时标注条文 id；原文未涉及处明确说明"书中无此法"，不得引入后世六壬规则。
user:   【课盘】{ZsljChart JSON}
        【命中条文】{refs 检索出的原文片段数组}
        【占问】{用户问题，如"占待人何时至"}
```

检索策略：不需要向量库——按**规则命中**（课体卦 id、占类章节、盘面出现的天将/月将）从 `data/book/*.json` 精确取条文即可，全书仅 4.5 万字，规则检索比语义检索更准。

## 10. 风险与决策点

1. **原文歧义**（§5.3 已定处理原则）——实现期逐条记录到 `docs/plan/textual-issues.md`；
2. **liuren-ts-lib 非商用声明**——只作可选对照引擎、动态加载；自研引擎仅依赖 MIT 的 tyme4ts，项目整体不受传染；
3. **月将模式**影响所有盘面——默认中气过宫（天文正确），复现古课例用字面模式，UI 显著标注当前模式；
4. **原站版权**——原文为公有领域；浦木裕的补注/旁注属其劳动成果，站内仅作研究引用并标明出处，不作为产品文案。
