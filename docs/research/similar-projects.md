# 六壬算法与同类项目调研（2026-07-05）

> 结论先行：**互联网上不存在《占事略決》的现成实现**（GitHub 搜索 `占事略決` / `senji ryakketsu` 均无有效结果）。
> 但"通行体系"大六壬的历法与排盘基础设施非常成熟，可以直接复用；**古法差异部分（约 300~500 行核心逻辑）需要我们按原文自研**。

## 一、可直接复用的库

### tyme4ts（首选历法底座）
- **仓库**：https://github.com/6tail/tyme4ts ·（lunar-javascript 作者 6tail 的升级版）
- **许可证**：MIT ✅（可商用、可自由分发）
- 提供：公历/农历互转、四柱干支（含早晚子时/立春分界选项）、节气/中气、旬空、神煞基础
- 引擎所需的"月将（太阳过宫）"可由中气推得；`liuren-ts-lib` 也是基于它实现的
- **用途**：react-zslj 自研引擎的唯一硬依赖

### liuren-ts-lib（通行体系大六壬，做"对照引擎"）
- **仓库**：https://github.com/look-fate/liuren-ts-lib · npm `liuren-ts-lib@3.0.0`（2026-04 仍活跃）
- **许可证**：⚠️ Apache-2.0，但 README 附加"仅供个人学习研究、禁止商用"声明（两者矛盾，从严理解按非商用对待）
- 提供：天地盘 / 四课 / 三传（九宗门）/ 课体名 / 遁干·初建·伏建 / 建除 / 神煞 / 阴阳贵人 / 虚岁流年 / 四柱⇄公历互转
- 已知缺陷：`getLiuRenBySiZhu` 抛错（react-liuren 已在适配器本地反推日期绕过，代码可直接搬）
- **用途**：作为"通行体系"对照引擎（可选依赖），与占事略決古法并排显示差异

### mingyu-core
- **仓库**：https://github.com/Brhiza/mingyu（176★）· npm `mingyu-core`
- **许可证**：MIT ✅
- 特点：八字/紫微/奇门/大六壬/小六壬等一站式排盘，**输出结构化提示词与数据**，提供公开 API 与 MCP Server——其"排盘数据 → AI 提示词"的思路与本项目 AI 分析方向一致
- **用途**：第二对照引擎（react-liuren 已有现成适配器）；AI 提示词结构可借鉴

## 四、《占事略決》古法与通行体系的已知差异（自研引擎的价值点）

初读原文（第一~三章、第廿六章）即可确认以下分歧，通行库都无法直接给出"晴明古法"的结果：

| 规则点 | 占事略決（983 年前后） | 通行体系（明清《六壬大全》系） |
|---|---|---|
| 涉害课取传 | 只按"加孟为深、加仲为半、加季为浅" | 沿地盘归本家计克数，复等再取孟仲季 |
| 涉害俱等 | "先举"：日先辰后、阳先阴后（第二章第四条） | 缀瑕课：取干上神/支上神 |
| 别责课 | **不存在**（课用九法无此门） | 九宗门含别责 |
| 伏吟三传 | 用神→其刑为中传→其冲为末传 | 自任/自信/杜传，迤逦相刑等细则 |
| 返吟无克 | 罡日取日冲、柔日取辰冲；丁己辛之丑未日专条（太一临亥用/徵明临巳用）；"传辰上，终日上" | 井栏射课取驿马发用，中末取支上/干上 |
| 八专 | 罡日自日上神顺数三神，柔日自辰上阴神逆数三神，"终日辰上" | 阳日干上阳神顺数三，阴日第四课上神逆数三；中末皆干上神 |
| 贵人昼夜分界 | **寅~酉为旦、戌~丑为暮**（第三章） | 卯~申为昼、酉~寅为夜 |
| 贵人歌诀 | 甲戊庚→旦丑暮未；乙己→旦子暮申；丙丁→旦亥暮酉；（六）辛→旦午暮寅；壬癸→旦巳暮卯（古歌"甲戊庚牛羊"系） | 多用"甲羊戊庚牛"改订歌诀（唐宋以后） |
| 天将名号 | 天一、腾虵、勾陣、大陰、大裳、徵明（登明）、勝先（胜光）、太一（太乙） | 贵人、腾蛇、勾陈、太阴、太常、登明、胜光 |
| 月将 | 正月徵明~十二月神后按月配将（第五章字面）；补注称"月将=太阳宫" | 按中气太阳过宫换将 |
| 课体系统 | **卅六卦**（第廿六章，含气类物/新故等占事略決特有卦） | 课体六十余种（元首/重审/知一/涉害…） |

> 注：原文若干处文义存疑（八专"终日辰上"、返吟三传细则、贵人顺逆"背天门向地户"的推广），引擎实现时需按字面实现 + 提供选项开关，并在 UI 标注校勘说明。详见项目方案。

## 五、结论

1. **历法底座用 tyme4ts（MIT）**，避开 liuren-ts-lib 的非商用声明对自研代码的传染——自研引擎只依赖 tyme4ts。
2. **占事略決古法引擎自研**，以 `docs/book/senji-ryakketsu.md` 为唯一规范，每条规则在代码中注明章节出处；书中卅六卦章附带的 30+ 条"假令"课例（如"元首：正月甲子日寅时占"）直接充当黄金测试用例。
3. **liuren-ts-lib / mingyu-core 作为可选"对照引擎"**，沿用 react-liuren 的注册表/适配器模式，UI 中并排展示古法与通行法三传差异并引用原文解释。
